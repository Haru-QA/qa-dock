QA Process Work Flow

과제 기획부터 릴리즈까지 6단계 QA 프로세스

1 기획/준비
→
2 일정/티켓
→
3 TC 설계
→
4 Dev QA
→
5 배포
→
6 릴리즈 후

Phase 1: 기획 & 과제 준비

PRD 작성 → 과제 리뷰 미팅 → 담당자 지정 → 배포 스레드 생성

단계활동담당도구산출물
1-1 PRD 작성 기획팀 Confluence PRD 문서
1-2 프로덕트 리더에게 기획 1차 공유 (선택) 기획팀 Figma -
1-3 후보 과제 리뷰 미팅 (배포 1주전 수요일) 전체 Confluence 후보 과제 확정
1-4 유관 부서 대상 과제 리뷰 미팅 기획팀 - -
1-5 디자인/서버/개발 담당자 지정 리더 JiraExcel -
1-6 디자인 가이드 공유 디자인팀 Figma 디자인 가이드
1-7 #그립-프로덕트--배포진행 Thread 생성 QA 리드 Slack 배포 스레드

Phase 2: 일정 산정 & 티켓 생성

상세 리뷰 → 고객가치/에픽 티켓 생성 → QA 할당 → 일정 확정

단계활동담당도구산출물
2-1 상세 과제 리뷰 미팅 (배포 주간 목/금) 기획팀 Confluence -
2-2 고객가치 티켓 생성 (Assignee, Fixed version, Priority) 기획자 Jira (GRIPPGM) 고객가치 티켓
2-3 과제별 Slack Thread/채널 생성 기획팀 Slack 과제 스레드
2-4 도메인별 에픽 티켓 생성 (Start/Due date) 개발팀 Jira (WEB/SVR/AND/iOS) 개발 에픽
2-5 QA 에픽 티켓 생성 QA 리드 Jira (GRIPPE) QA 에픽
2-6 개발 일정 산정 완료 -> Slack 과제 스레드에 남김 개발팀 Slack -
2-7 담당 QA 할당 + QA시작일/종료일 업데이트 QA 리드 Jira -
2-8 TC 에픽 티켓 생성 + 필수 항목 업데이트 담당 QA Jira (GRIPPE) TC 에픽
2-9 일정 산정 완료 -> #product-leads 공유 QA 리드 Slack -
2-10 Jira Release Tag 및 Timeline 관리 QA 리드 Jira -

Phase 3: TC 설계

기획서 + 디자인 피그마 기반 TC 작성 → 리뷰 → 확정

단계활동담당도구산출물
3-1 기획서 + 디자인 피그마 기반 TC 작성 담당 QA xmind TC (xmind)
3-2 QA Weekly에서 TC 리뷰 QA팀 - 리뷰 피드백
3-3 TC 확정 및 수정 반영 담당 QA xmind TC 확정본

Phase 4: 개발 & QA (Dev QA)

개발 완료 → Dev QA 수행 → 버그 등록/수정 → 리그레션

단계활동담당도구산출물
4-1 개발 진행 개발팀 - -
4-2 개발 완료 -> 에픽 "QA중" 전환 개발팀 Jira -
4-3 Dev QA (개발환경 테스트 수행) 담당 QA 개발환경 테스트 결과
4-4 버그 발견 -> Jira 버그 등록 담당 QA Jira 버그 티켓
4-5 개발자 버그 수정 (대기중->개발중->머지됨->QA중) 개발팀 Jira -
4-6 수정 확인 -> 리그레션 테스트 담당 QA - -
4-7 수정 완료 -> "완료" 처리 / 미수정 -> "대기중" 재전환 담당 QA Jira -

Phase 5: 배포

정기배포 (앱, 2주 주기) + 비정기배포 (서버+웹)

단계활동담당도구요일
정기배포 (앱, 2주 주기)
5-1 개발환경 테스트 완료 QA팀 - 월요일
5-2 서버 운영 배포 개발팀 - 화요일 오후
5-3 세니티 테스트 + 운영 리그레션 QA팀 운영환경 화요일
5-4 앱 크래쉬 로그 확인 QA팀 - 화요일
5-5 QA 티켓 + 도메인 에픽 티켓 닫기 QA팀 Jira 화요일
5-6 배포 스레드에 앱 Sign-off QA팀 Slack 화요일
5-7 앱스토어 심사 요청 개발팀 App StorePlay Store 화~수
비정기배포 (서버+웹)
5-8 개발환경 테스트 완료 + 티켓 닫기 QA팀 Jira 화요일
5-9 서버/프론트 운영 배포 개발팀 - 수요일
5-10 세니티 테스트 + 웹 자동화 실행 QA팀 Selenium 수요일
배포 후 공통
5-11 배포 완료 공유 + 모니터링 개발팀 Slack -
5-12 리그레션 테스트 완료 공유 QA팀 Slack -
5-13 과제 고객가치 티켓 정리 기획팀 Jira -
5-14 릴리스 노트 공유 기획개발 Slack -

Phase 6: 릴리즈 후

앱스토어 출시 → 모니터링 → 100% 롤아웃 → 라이브 버그 대응

단계활동담당도구
6-1 앱스토어 출시 + 점진 배포 개발팀 App StorePlay Store
6-2 모니터링 (크래쉬, VOC) QA개발 Slack모니터링
6-3 100% 롤아웃 개발팀 -
6-4 라이브 버그 -> Slack 공유 -> QA 원인 조사 -> Jira 등록 -> 핫픽스 QA개발 SlackJira

Bug Status Flow

대기중 → 개발중 → 리뷰중 → 머지됨 → QA중 → 완료
미수정 시: QA중 -> 대기중 재전환  |  오탐: QA중 -> 버그아님  |  보류: 대기중 -> 백로그

Bug Priority

등급설명예시
Blocker앱 사용 불가App Crash, Login Fail
Critical심각한 기능 문제결제 오류, 방송 오류
Major눈에 띄는 문제기대 결과와 다름
Minor사소한 문제텍스트 버그, 낮은 재현율
Trivial거의 문제 없음유사 표현 차이
