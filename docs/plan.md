# 요구사항 정의 — QA Dock (팀 대시보드)

> 상위 프로젝트: improve-productivity-qa > H. 팀 대시보드
> 시작일: 2026-03-26
> 최종 업데이트: 2026-03-26

---

## 1. 프로젝트 목적

QA팀의 모든 업무 현황을 **단일 대시보드**로 통합하여:
- 팀 리드가 전체 QA 상황을 실시간 파악
- 담당 QA가 자신의 업무 우선순위를 한눈에 확인
- 이해관계자(기획, 개발)에게 QA 진행 상황을 투명하게 공유

---

## 2. 대시보드 구성

### 전체 페이지 레이아웃 (단일 페이지, 사이드바 없음)
> 참고: grip-dashboard 레이아웃 구조 차용

```
┌─────────────────────────────────────────────────────────┐
│ 헤더 (QA Dock 타이틀, 최종 업데이트, 총 QA 건수)       │
├─────────────────────────────────────────────────────────┤
│ 요약 카드 (QA 완료 / 진행 중 / 대기 건수)               │
├─────────────────────────────────────────────────────────┤
│ 🔄 QA 워크플로우 요약 (6단계 플로우)                    │
├─────────────────────────────────────────────────────────┤
│ 📅 배포 일정 (월별 타임라인)                             │
├─────────────────────────────────────────────────────────┤
│ ⚙️ QA 상황판 (4단계 카드 → 클릭 시 하단 테이블)         │
├─────────────────────────────────────────────────────────┤
│ 🚀 최근 QA 완료 목록                                    │
├──────────────┬──────────────┬───────────────────────────┤
│ 📊 상태별    │ 🎯 우선순위별 │ 👤 담당 QA별 리소스       │
│    현황      │    현황       │    현황                   │
└──────────────┴──────────────┴───────────────────────────┘
```

- 반응형 그리드 레이아웃 (사이드바 없음)
- 모달 팝업으로 상세 데이터 확인

### 2.1 QA 상황판 (메인 화면)

#### 2.1.1 QA 워크플로우 요약 (최상단)
- QA 프로세스 6단계를 간략한 플로우 형태로 표시
  - 기획/준비 → 일정산정/티켓 → TC설계 → Dev QA → 배포 → 릴리즈 후
- 현재 활성화된 프로젝트들이 어느 단계에 몇 건 있는지 숫자 표시
- 각 단계 클릭 시 해당 단계의 프로젝트 목록으로 이동

#### 2.1.2 4단계 상황판 카드
> 참고: grip-dashboard 개발 부킹 현황 카드 구조 차용

QA 업무의 핵심 흐름을 4개 카드로 표시:

| 단계 | 색상 | 설명 | Jira 상태 매핑 (예시) |
|------|------|------|---------------------|
| **QA 필요** | 빨강 (#ef4444) | 개발 완료되어 QA 대기 중인 건 | `개발완료`, `QA대기` |
| **QA 리뷰** | 주황 (#f97316) | TC 설계/리뷰 진행 중인 건 | `TC작성중`, `TC리뷰` |
| **QA 진행** | 파랑 (#3b82f6) | 실제 테스트 수행 중인 건 | `QA중`, `Dev QA` |
| **QA 완료** | 초록 (#22c55e) | QA 종료된 건 (배포 대기 포함) | `QA완료`, `배포대기` |

각 카드 구성:
- **건수(숫자)**: 큰 폰트로 강조
- **단계 라벨**: 카드 제목
- **부제 설명**: 해당 단계의 간략 설명
- **색상 배경**: 단계별 고유 색상
- 카드 클릭 → 하단에 해당 단계의 **상세 리스트 테이블** 펼침

#### 2.1.3 상세 리스트 (하단)
카드 클릭 시 하단에 테이블 형태로 해당 단계의 전체 목록을 표시.
> 참고: grip-dashboard의 개발 부킹 현황 리스트 구조를 QA 맥락에 맞게 변환

| # | 필드 | 표시 형식 | 설명 | 정렬 |
|---|------|----------|------|------|
| 1 | **도메인** | 텍스트 | 프로덕트/서비스 도메인 분류 | O (가나다순) |
| 2 | **과제명** | 텍스트 (말줄임) | 고객가치 티켓명, 최대 너비 제한 + 말줄임표 | - |
| 3 | **우선순위** | 컬러 배지 | Blocker(빨강) / Critical(주황) / Major(노랑) / Minor(초록) / Trivial(회색) | O (심각도순) |
| 4 | **담당 QA** | 텍스트 | 할당된 QA 담당자 | O (가나다순) |
| 5 | **상태** | 컬러 배지 | 현재 Jira 상태 (STATUS_COLOR 매핑) | O (상태순) |
| 6 | **목표일** | 텍스트 (YY.MM.DD) | QA 종료 목표일 (Due Date) | O (날짜순) |
| 7 | **Jira** | 링크 버튼 | "GRIPPE-XXXX ↗" 클릭 시 Jira 새 탭 오픈 | - |
| 8 | **완료처리** | 버튼 | "QA 진행" 단계에서만 표시, QA 완료 처리 | - |

##### 상태값 목록 (STATUS_COLOR 매핑)
QA 프로세스에 맞는 상태값 정의:

| 상태 | 색상 | 설명 |
|------|------|------|
| QA 대기 | 회색 | 개발 완료 후 QA 시작 전 |
| TC 작성중 | 보라 | TC 설계 진행 중 |
| TC 리뷰 | 남색 | TC 리뷰 대기/진행 중 |
| QA 중 | 파랑 | Dev QA 수행 중 |
| 버그 수정 대기 | 주황 | 버그 등록 → 개발자 수정 대기 |
| QA 재검증 | 청록 | 버그 수정 후 재검증 중 |
| QA 완료 | 초록 | QA Sign-off 완료 |
| 배포 대기 | 연두 | QA 완료 → 배포 대기 |
| 릴리스 | 진초록 | 배포 완료 |
| 보류 | 회색 | 일시 보류 |
| 대기 | 연회색 | 의사결정 대기 |

##### 테이블 인터랙션
- **컬럼 헤더 클릭**: 해당 컬럼 기준 오름/내림차순 정렬 토글
- **행 호버**: 배경색 하이라이트
- **행 완료 처리 시**: 배경색 초록(#f0fdf4)으로 변경
- **"QA 완료" 카드**: 완료 버튼 미표시, 릴리스 상태 건 제외

### 2.2 통계/분석 대시보드 (하단 3열 그리드)
> 참고: grip-dashboard 하단 3열 그리드 레이아웃 차용

단일 페이지 내 하단에 3열 그리드로 배치:

#### 2.2.1 상태별 현황
- 도넛 차트 + 범례
- 전체 QA 티켓의 상태별 비율 (STATUS_COLOR 기준)
- 클릭 시 해당 상태 건 목록 모달 표시

#### 2.2.2 우선순위별 현황
- 가로 막대 차트
- Blocker/Critical/Major/Minor/Trivial 배지 색상 동일 적용
- 미해결 건만 필터 옵션

#### 2.2.3 담당 QA별 리소스 현황
- 스택 바 차트
- 담당자별 진행 중 / 대기 중 건수
- 과부하 경고 (임계치 초과 시 하이라이트)

#### 2.2.4 배포 일정 (상황판 상단)
> 참고: grip-dashboard 월별 배포 일정 섹션 차용
- 월별 배포 타임라인 표시
- 각 배포 주기에 포함된 QA 건수
- 현재 배포 주기 하이라이트

#### 2.2.5 기간별 추이 (추후)
- 라인 차트 (주간/월간 토글)
- QA 요청 건수 vs QA 완료 건수 추이
- 평균 QA 소요 기간 (리드 타임)

#### 2.2.6 버그 지표 (추후)
- 발견율: 프로젝트당 평균 버그 수
- 해결율: 등록 대비 완료 비율
- 심각도 분포 추이

---

### 2.3 도구 연동 대시보드

#### 2.3.1 정적 분석 현황 (D. PRD 정적분석 봇 연동)
- 최근 분석 실행 목록
- 프로젝트별 분석 결과 요약 (발견 이슈 수, 심각도)
- 분석 리포트 링크

#### 2.3.2 TC 설계 현황 (E. PRD TC 생성 봇 연동)
- TC 자동 생성 현황 (생성 건수, 달성률)
- 프로젝트별 TC 커버리지
- TC 리포트 링크 (xmind, HTML)

#### 2.3.3 CI/CD 현황 (C. CI/CD 자동화 연동)
- 최근 빌드/배포 상태
- 자동화 테스트 실행 결과
- 실패 알림

## 3. 데이터 모델

### 3.1 주요 데이터 소스

| 소스 | 데이터 | 갱신 주기 |
|------|--------|----------|
| Jira REST API | QA/버그 티켓, 에픽, 상태, 담당자 | 실시간 or 5분 폴링 |
| Jira REST API | 고객가치(GRIPPGM) 티켓 | 실시간 or 5분 폴링 |
| 정적분석 결과 | 분석 리포트 메타데이터 | 분석 실행 시 |
| TC 자동화 결과 | TC 리포트 메타데이터 | TC 생성 시 |
| CI/CD | 빌드/배포 상태 | 이벤트 기반 |

### 3.2 Jira 프로젝트 매핑

```
GRIPPGM  → 고객가치 (최상위)
GRIPPE   → QA 에픽, TC 에픽, 버그 티켓
GRIPWEB  → 웹 프론트 개발 에픽
GRIPSVR  → 서버 개발 에픽
GRIPAND  → 안드로이드 개발 에픽
GRIPIOS  → iOS 개발 에픽
```

### 3.3 상태 매핑 (확정 필요)
> Jira 워크플로우의 실제 상태값을 조사하여 4단계 카드에 매핑해야 함.
> 상위 프로젝트의 `qa-process-asis.md` 참조.

## 5. 단계별 구현 계획

### Phase 1: 상황판 MVP
- QA 4단계 상황판 카드 + 상세 리스트
- Jira API 연동 (GRIPPE 프로젝트)
- 기본 필터 (담당자, 우선순위)

### Phase 2: 워크플로우 + 통계
- QA 워크플로우 요약 표시
- 상태별/우선순위별/담당자별 차트
- 기간별 추이 차트

### Phase 3: 도구 연동
- 정적분석 결과 연동
- TC 설계 현황 연동
- CI/CD 현황 연동

### Phase 4: 고도화
- 실시간 갱신 (WebSocket or SSE)
- 알림 연동 (Slack)
- 개인화 (즐겨찾기, 커스텀 뷰)
- 모바일 반응형
